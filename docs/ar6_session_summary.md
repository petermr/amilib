## Processing Pipeline Review

### Current Pipeline (from `test/ipcc_classes.py`)

1. **Download** → `gatsby_raw.html`
2. **Clean** → `de_gatsby.html` (removes navigation, tooltips, etc.)
3. **Add IDs** → `html_with_ids.html` (adds paragraph and section IDs)

### ID Generation Logic

**Section IDs:**
- Extracted from container divs: `div[@class="h2-container"][@id="3.1.2"]`
- Pattern: `\d+(\\.\d+)*` or special sections like `executive-summary`, `references`

**Paragraph IDs:**
- Format: `{section_id}_p{index}` (e.g., `3.1.2_p1`, `3.1.2_p2`)
- Generated by `IPCCGatsby.create_and_add_id()`
- Requires parent container to have ID

### Issues Identified

1. **Paragraphs without parent section IDs** don't get IDs
2. **Duplicate IDs** occur when multiple containers share same section number
3. **Front matter sections** (authors, citation) may not follow standard ID patterns
